Baum ist ein aus Knoten und Kanten bestehender Graph, besitzt dabei immer genau eine Wurzel und enthält keine Zyklen. Außer der Wurzel sind alle Knoten durch genau eine Kante (Pfad) mit einem Elternknoten (Vorgänger) verbunden und wird als dessen Kindknoten (Nachfolger) bezeichnet. Knoten ohne Nachfolger heißen Blatt. Alle Knoten mit Nachfolgern werden als innere Knoten bezeichnet. Bäume wachsen in diesem Kontext „unnatürlich“ von oben (Wurzel) nach unten (Blatt).
   Binärer Baum ist ein geordneter Baum, bei dem jeder Knoten maximal zwei Nachfolger hat: einen linken - k.left und einen rechten - k.right. Unter der Ordnung eines binären Baums versteht man die maximale Anzahl von Kindknoten pro Knoten.
   Binärer Suchbaum ist ein geordneter binärer Baum, dessen Knoten Schlüsselwerte enthalten. Über diese Werte erfolgt die Suche im Baum. Alle Knoten k enthalten einen Schlüsselwert k.data. Auf die Kindknoten wird über k.left und k.right zugegriffen. Die Höhe des Baums entspricht dem längsten Pfad von der Wurzel zum tiefesten Blatt plus 1.

Für jeden Knoten k eines binären Suchbaums gilt dabei:
  • Alle Schlüsselwerte im linken Teilbaum k.left sind kleiner als der Schlüsselwert k.data.
  • Alle Schlüsselwerte im rechten Teilbaum k.right sind grösser als der Schlüsselwert k.data
public class BinarySearchTree {
private Node insertNodeRecursive(Node head, int data) {
if (head == null) {
return new Node(data);
} else if (data > head.data) {
head.right = insertNodeRecursive(head.right, data);
} else if (data <= head.data) {
head.left = insertNodeRecursive(head.left, data);
}
return head;
}
}
